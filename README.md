#  Student Portfolio Website – Real-World Production Project
This project is a custom-built portfolio website created to document my daughter’s academic journey, leadership roles, and milestone events—including her Senior Capstone Showcase.
Unlike a classroom demo, this was developed as a **real-world production project**, with continuous improvements focused on performance, usability, and long-term maintainability.
---
##  Live Site  
 https://software-me.github.io/Loreine-Portfolio/  
---
##  Project Purpose  
- Provide a polished, accessible online portfolio for academic achievements and activities  
- Organize photos and videos into structured, easy-to-navigate sections  
- Enable long-term updates through a scalable, data-driven approach  
- Demonstrate real-world deployment using GitHub Pages  
---
##  What I Built  
- Multi-page static website using **HTML, CSS, and JavaScript**  
- Reusable navigation and layout components across all pages  
- Dedicated media sections for:
  - PSST events (photos & videos)  
  - Senior Capstone Showcase  
  - YouTube-ready embedded content  
---
##  Technical Highlights  
###  Structured & Semantic HTML  
- Clean, semantic layouts across key sections (Home, About, Goals, Activities, Reflection, Contact)  
- Dedicated event pages for media-heavy content  
- Accessible structure for readability and maintainability  
###  Scalable CSS System  
- Centralized styling in `css/styles.css`  
- Reusable UI components (cards, galleries, typography, spacing)  
- Responsive design for desktop and mobile  
- Custom layouts for media-rich pages  
###  JavaScript Architecture  
- `js/main.js` handles shared UI behavior and navigation  
- Modular scripts (`psst-gallery.js`, `senior-showcase.js`) for:
  - Dynamic media rendering  
  - Section-based loading  
  - Performance-focused video handling  
###  Data-Driven Media Management  
- Media content stored in structured JS data files (e.g. `assets/data/*.js`)  
- Each item includes:
  - `file`, `title`, `description`  
- Enables easy updates without modifying page structure  
---
##  Media & Asset Organization  
- Images are organized by event (`psst`, `senior-showcase`, etc.)  
- Media metadata is separated into dedicated data files  
- JavaScript dynamically renders content from structured sources  
###  Video Handling  
- Video content is hosted externally (e.g., YouTube) and embedded into the site  
- Large media files are excluded from version control to maintain repository efficiency
